# 06｜视觉系统、实现约束与 vibe coding 总提示词

## 1. 视觉关键词

```text
星空
数字花园
个人宇宙
记忆地图
毛玻璃
柔和渐变
圆角卡片
轻量动效
```

## 3. 字体建议

中文：

```text
Noto Sans SC / HarmonyOS Sans / system-ui
```

英文和数字：

```text
Inter / Geist / system-ui
```

## 4. 组件规范

### Card

```text
圆角：20px - 28px
背景：半透明 / 毛玻璃
边框：1px 半透明
阴影：柔和
hover：上浮 4px + 边框变亮
```

### Button

主要按钮：

```text
渐变背景
圆角 pill
轻微光晕
hover 时亮度提升
```

次要按钮：

```text
透明背景
半透明边框
hover 时背景变亮
```

### Tag

用途：身份、技术栈、文章分类、旅行地点、游戏类型。

```text
小圆角
半透明背景
轻微边框
支持不同颜色，但不要过多
```

### Timeline

用途：教育经历、旅行轨迹、网站更新。

```text
左侧发光节点
中间连线
右侧内容卡片
当前节点高亮
```

### ProfileCard

字段：

```text
头像
昵称
一句话介绍
身份标签
联系方式
按钮
```

约束：

- `/about` 桌面端 sticky。
- 移动端放在页面顶部。
- 头像不要变形。

---

## 5. 动效约束

允许：

```text
淡入
轻微上浮
星点慢速漂浮
按钮 hover 光晕
卡片 hover
锚点平滑滚动
```

避免：

```text
大量旋转
频繁闪烁
过重 3D
页面加载很慢
滚动劫持
影响阅读的背景动画
```

性能：

- 动画优先使用 transform 和 opacity。
- 图片懒加载。
- 移动端减少复杂动效。

---

## 6. 响应式约束

桌面端：

```text
最大内容宽度：1120px - 1280px
首页 Hero 可双栏
/about 可左侧 sticky + 右侧内容
/world 可使用地图节点 + 卡片网格
```

手机端：

```text
全部单列
导航折叠
卡片宽度 100%
Hero 文案和头像上下排列
锚点导航横向滚动
```

---

## 7. 内容与隐私约束

上线前必须替换：

```text
测试标题
无意义文章内容
空摘要
默认图片描述
无效联系方式
```

推荐占位符：

```text
{{avatarUrl}}
{{email}}
{{githubUrl}}
{{hometown}}
{{schoolName}}
{{techStack}}
{{travelPlaces}}
{{games}}
```

隐私：

- 不公开详细住址。
- 微信、QQ 使用二维码弹窗或“点击显示”。
- 学校信息如果不想完全公开，可以写阶段和方向，不写精确班级。

## 10. 总提示词，可直接交给 vibe coding

```text
请基于现有 lqpp.shop 个人博客进行一次较大规模改版。当前网站已有首页、文章、相册、3D 星空相册和后台入口。请保留文章和相册等已有能力，但将整体信息架构升级为“LQPP World”个人数字花园。

需要实现或规划以下页面：
1. 首页：个人世界入口页，包含 Hero、头像个人卡、四个入口卡片、最新文章、精选相册、技术栈预览和 Footer。
2. /about：关于我页面，包含头像、联系方式、个人介绍、教育经历时间线、技术栈、个人特点。
3. /world：我的世界页面，包含家乡、学校、旅行探索、游戏宇宙和我的特点，整体像一张个人世界地图。
4. /articles 和 /posts/[slug]：优化文章列表、分类、搜索、文章详情排版、摘要和阅读体验。
5. /photos：改造成“记忆星图”，保留普通相册和 3D 星空相册，增加分类和照片信息。
6. 可选 /guestbook、/projects、/timeline，用于留言、技术作品和成长时间线。

整体视觉风格：星空、数字花园、个人宇宙、记忆地图、毛玻璃、柔和渐变、圆角卡片、轻量动效。不要做成企业官网或普通模板博客。导航最多保留 6 个公开入口：首页、文章、相册、我的世界、关于我、留言。后台入口不要放在主导航中。

实现要求：响应式布局，移动端单列，桌面端可双栏；个人信息和内容数据抽离到配置文件；所有卡片和标签组件化；图片懒加载；动画轻量；联系方式注意隐私；上线前替换测试文章、空摘要、默认图片描述和无效社交链接。
```
