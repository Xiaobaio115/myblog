# Luna Notes 首页改版 Vibe Coding 实现规范

> 目标：在不修改现有接口、路由、字段、后台逻辑的前提下，对 Luna Notes 博客首页进行视觉升级。设计风格保持现有网站结构，但升级为 Linear 感的现代粉色主题，并支持 Light / Dark 主题切换。

---

## 0. 给 AI 编码工具的总指令

```text
请基于当前已有 Luna Notes 博客项目进行首页 UI 改版。

这是一次纯前端视觉升级，请严格遵守：
1. 不要修改现有接口 URL。
2. 不要修改请求方式。
3. 不要修改数据字段名。
4. 不要修改返回数据结构。
5. 不要修改后台管理相关逻辑。
6. 不要改乱现有页面路由。
7. 不要重写业务逻辑。
8. 不要新增必须依赖后端的新字段。

请把现有页面视为已经跑通的数据骨架，只升级 UI 皮肤、布局、组件样式、主题系统和轻微交互。

目标风格：
- 参考 Linear 的现代极简产品感。
- 使用粉色作为品牌主色。
- 支持 Light / Dark 主题。
- 卡片使用细边框、圆角、柔和阴影。
- 保持博客的温柔内容感，不要改成复杂后台系统。

首页保留现有模块顺序：
Header → Hero → Stats → Featured Article → Latest Articles → Featured Albums → Footer。

请优先采用组件化方式实现，并保证响应式。
```

# 3. 设计风格说明

## 3.1 关键词

```text

现代极简

轻边框
柔和阴影
```

* 文字层级要明显。
* 暗色模式不要纯黑，使用深灰黑。
* 保留现有网站“个人博客”的内容气质。

# 5. 组件任务清单

## 5.1 Task 1：全局主题系统

### 目标

实现 Light / Dark 主题切换，不影响任何业务逻辑。

### 具体要求

* 使用 `data-theme="light"` / `data-theme="dark"` 控制主题。
* 主题状态保存到 `localStorage`。
* 初次进入优先读取本地存储。
* 如果没有本地存储，可读取系统偏好 `prefers-color-scheme`。
* 切换主题时只改变 CSS variables，不重新请求数据。

### 建议实现

```text
ThemeProvider
ThemeToggle
useTheme
```

### 验收标准

* 点击按钮可以切换亮色/暗色。
* 刷新页面后主题保持。
* 切换主题不会导致文章/相册接口重新设计或字段变化。
* Light / Dark 下所有文字可读。

# 12. Tailwind 实现建议

如果项目使用 Tailwind，建议将 CSS variables 接到 Tailwind theme：

```ts
// tailwind.config.ts
export default {
  theme: {
    extend: {
      colors: {
        bg: 'var(--bg)',
        surface: 'var(--surface)',
        'surface-soft': 'var(--surface-soft)',
        text: 'var(--text)',
        muted: 'var(--text-muted)',
        border: 'var(--border)',
        primary: 'var(--primary)',
        'primary-2': 'var(--primary-2)',
        'primary-3': 'var(--primary-3)',
      },
      borderRadius: {
        card: 'var(--radius-md)',
      },
      boxShadow: {
        card: 'var(--shadow-card)',
        hover: 'var(--shadow-hover)',
      },
    },
  },
};
```
